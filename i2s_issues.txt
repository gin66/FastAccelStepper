- The i2s buffer must be a single array of contiguous bytes (3 * buffer size) - buffer 0/1/2
- On dma start, the buffers 0/1/2 are enqueued and busy_block to be set to 0
- The dma callback will enqueue to DMA the busy_block and then advances busy_block = (busy_block+1)%3. Then fill_blocks of all steppers are called.
- fill_i2s_buffer maintains a single tick pointer (tp in 16MHz). 
  With 8 Mhz BCLK, every bit is 2 ticks (@16MHz) and each frame is 64ticks .... using 16bit, with 8bit is half
  The frame pointed to by tp is frame_index = (tp / 2 ticks) / (32 ticks/frame)
  In single stepper mode we get better resolution:
           j = (tp / 2)%32 => bit index  this index may take into account the byte organisation in each frame
- fill_is_buffer loops:
  - tp = tp % (3*buffer_size*64 ticks)
  - if tp is inside busy_block: advance to beginning of next next buffer
  - if high_ticks/low_ticks = 0, then next cmd is read:
    if cmd.steps = 0, then high_ticks = 0 and low_ticks = cmd_ticks. Advance queue read pointer
    if cmd.steps >= 1, then high_ticks = 64 and low_ticks = cmd.ticks-64. if cmd.steps-- == 0, then advance queue read pointer
  - fill_i2s_buffer check if there are still high_ticks/low_ticks to be updated. If no, continue loop
  - while high_ticks > 0: add bit to buffer, record bit in performed steps, increase tp by bit time.
          if tp is now in busy_block, exit the loop.
  - if low_ticks > 0: calculate free_ticks from tp till busy_block.
          if free_ticks > low_ticks: tp += low_ticks. low_ticks = 0. continue loop
          if free_ticks <= low_ticks: tp += free_ticks. low_ticks -= free_ticks. exit loop
